APPLICATION PURPOSE
================================================================================

EluSEEDate is a mobile navigation assistance application for visually impaired
users. It combines computer vision, voice recognition, and AI prediction to
provide real-time navigation guidance through:
- Object detection (obstacles in path)
- Turn prediction (user's intended direction)
- Voice command processing (hands-free control)
- Text-to-speech feedback (audio guidance)

================================================================================
SYSTEM ARCHITECTURE
================================================================================

The application uses a multi-model architecture with four core components:

1. VOSK Speech Recognition
2. Text-to-Speech (TTS)
3. YOLO Object Detection
4. ConvLSTM Turn Prediction (with/without Intent)

All components work in parallel to provide comprehensive navigation assistance.

================================================================================
COMPONENT 1: VOSK SPEECH RECOGNITION
================================================================================

Library: react-native-vosk (v2.1.7)
Purpose: Offline voice command recognition for hands-free control

How It Works:
- Processes user's spoken commands locally on device
- No internet connection required (fully offline)
- Uses lightweight Vosk models optimized for mobile
- Recognizes navigation commands (e.g., "turn left", "where am I", etc.)

Model Location:
- assets/model-en-us/ (English language model)

Key Features:
- Real-time speech-to-text conversion
- Low latency (<100ms response time)
- Privacy-focused (all processing on-device)
- Works in noisy environments

Integration:
- User speaks command → VOSK processes → App executes action
- Commands trigger navigation changes, information requests, etc.

================================================================================
COMPONENT 2: TEXT-TO-SPEECH (TTS)
================================================================================

Library: expo-speech (v14.0.8)
Purpose: Audio feedback to communicate detection results to user

How It Works:
- Converts text information to spoken audio
- Announces detected obstacles (e.g., "car ahead", "person on right")
- Confirms user commands and system status
- Provides turn-by-turn navigation guidance

Key Features:
- Natural voice synthesis
- Adjustable speech rate and pitch
- Queue management for multiple announcements
- Language support for accessibility

Integration:
- YOLO detects object → TTS announces "car detected ahead"
- ConvLSTM predicts turn → TTS confirms "turning left"
- User command received → TTS acknowledges "turning navigation on"

================================================================================
COMPONENT 3: YOLO OBJECT DETECTION
================================================================================

Model: YOLOv12 TensorFlow Lite
Purpose: Real-time detection of obstacles in camera view

Input Specifications:
- Shape: (1, 3, 128, 128) BCHW format
- Resolution: 128x128 pixels
- Format: RGB channels
- Normalization: [0, 1] range

Output Specifications:
- Shape: (1, 84, 336)
- 84 values: 4 bbox coords + 80 class scores
- 336 potential detections

Detected Objects (80 COCO classes):
- People and animals (person, dog, cat, etc.)
- Vehicles (car, bicycle, motorcycle, bus, truck)
- Street furniture (traffic light, stop sign, bench)
- General urban obstacles

How It Works:
1. Camera captures frame
2. Frame resized to 128x128 and converted to RGB
3. YOLO model processes image
4. Outputs bounding boxes for detected objects
5. Non-Maximum Suppression filters duplicates
6. Results displayed as overlay on screen
7. TTS announces critical obstacles

Performance:
- Inference: 30-100ms per frame (device-dependent)
- Confidence threshold: 0.5 (50% minimum)
- IoU threshold: 0.45 for NMS

Model Location:
- assets/model/yolo.tflite

================================================================================
COMPONENT 4: CONVLSTM TURN PREDICTION
================================================================================

Model: Convolutional LSTM with Global Average Pooling
Purpose: Predict user's intended turn direction based on video sequence

The system has TWO operational modes:

Mode 1: ConvLSTM WITHOUT Intent
- Predicts turns based ONLY on video frames
- Pure computer vision approach
- Uses observed head/body movement patterns

Mode 2: ConvLSTM WITH Intent
- Combines video frames + user's spoken intent
- Intent encoded as additional 3 channels
- Improves prediction accuracy with explicit user input

Model Specifications:
- Input: (1, 20, 6, 128, 128) - Batch, Sequence, Channels, Height, Width
- 20 frames at 10 FPS = 1 second of video
- 6 channels: 3 RGB + 3 Intent (or 3 RGB + 3 zeros for no intent)
- Output: (1, 3) - Probabilities for [Front, Left, Right]

Architecture:
- 2 stacked ConvLSTM layers (64, 32 hidden dims)
- 3x3 convolutional kernels
- Global Average Pooling (mobile optimization)
- Dropout 0.5 for regularization
- Softmax output for class probabilities

How It Works:

WITHOUT Intent Mode:
1. Camera records 20 frames over 1 second
2. Frames resized to 128x128
3. Intent channels filled with zeros (no intent signal)
4. ConvLSTM processes sequence → predicts direction
5. System announces: "Predicting: Left 75%"

WITH Intent Mode:
1. User speaks: "I want to turn left"
2. VOSK recognizes command → Intent encoded
3. Camera records 20 frames over 1 second
4. Intent signal added to last 10 frames as extra channels
5. ConvLSTM processes video + intent → predicts direction
6. System announces: "Intent: Left, Predicting: Left 92%"

Intent Encoding:
- Three channels: [Left, Front, Right]
- Active intent = channel filled with 1.0
- Inactive = channel filled with 0.0
- Applied to last 10 frames of 20-frame sequence

Performance:
- Inference: 150-300ms (device-dependent)
- Accuracy: ~75% without intent, ~92% with intent
- Real-time operation at 10 FPS sampling

Model Locations:
- assets/model/convlstm.tflite (TFLite version)
- assets/model/convlstm.onnx (ONNX version, future use)

================================================================================
SYSTEM WORKFLOW
================================================================================

Typical User Experience:

1. User opens app → Camera activates
2. YOLO continuously scans for obstacles
3. TTS announces: "Car ahead, person on right"
4. User speaks: "I want to turn left" → VOSK recognizes
5. ConvLSTM WITH Intent starts recording frames
6. After 1 second: ConvLSTM predicts turn with intent
7. TTS announces: "Predicted: Left 92%"
8. System provides turn-by-turn guidance via TTS

Alternative (Without Voice Commands):
1. User opens app → Camera activates
2. YOLO scans for obstacles
3. ConvLSTM WITHOUT Intent continuously predicts direction
4. TTS announces: "Predicting: Front" or "Predicting: Left 75%"
5. User navigates based on predictions

================================================================================
AI MODEL PIPELINE
================================================================================

Parallel Processing:
- YOLO runs every frame (30-100ms per inference)
- ConvLSTM runs every 1 second (150-300ms per inference)
- VOSK listens continuously (processes when user speaks)
- TTS queue manages announcements (prevents overlap)

Data Flow:
Camera → Frame Buffer → [YOLO, ConvLSTM] → Results → TTS → User

YOLO Path:
Frame → Resize 128x128 → RGB conversion → YOLO → Detections → UI + TTS

ConvLSTM Path:
20 Frames → Resize 128x128 → RGB + Intent → ConvLSTM → Prediction → UI + TTS

VOSK Path:
Microphone → Audio buffer → VOSK → Text → Command parser → Intent encoder

================================================================================
MOBILE OPTIMIZATION
================================================================================

All models optimized for mobile deployment:

1. Resolution: 128x128 (low memory footprint)
2. Format: TensorFlow Lite (.tflite files)
3. Quantization: FP32 (accuracy vs size tradeoff)
4. GPU Acceleration: Enabled when available
5. Frame Sampling: 10-20 FPS (not full 30 FPS)
6. Global Average Pooling: Reduces model parameters

Model Sizes:
- YOLO: ~8-12 MB
- ConvLSTM: ~1.5 MB
- VOSK: ~40 MB (includes language model)

Total App Size: ~130-150 MB

================================================================================
KEY INNOVATION: INTENT INTEGRATION
================================================================================

Novel Contribution:
Traditional turn prediction uses only visual data. This system introduces
"intent channels" that encode the user's spoken goal directly into the neural
network input, creating a multimodal prediction system.

Benefits:
1. Higher accuracy when user provides intent (75% → 92%)
2. Graceful fallback to vision-only when no intent given
3. User maintains control over navigation decisions
4. Reduces false predictions in ambiguous scenarios

Implementation:
- Intent encoded as binary channels (one-hot encoding)
- Applied to last 50% of video sequence (10 of 20 frames)
- ConvLSTM learns temporal correlation between intent and action
- No model retraining needed for intent vs no-intent modes
