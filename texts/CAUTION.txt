CAUTION - IMPORTANT NOTES
=========================

NATIVE CODE & PREBUILD
-----------------------

⚠️ This project uses prebuilt native code (android/ folder exists)

When you modify app.json (plugins, permissions, splash, etc.):
1. Run: npx expo prebuild --clean
2. Rebuild the app: eas build --profile preview --platform android

If you skip prebuild, your app.json changes will NOT be applied.


VOSK MODEL REQUIREMENTS
------------------------

⚠️ Vosk model MUST be in assets/model-en-us/
- Folder name must start with "model-"
- Model files (am/, conf/, graph/, ivector/) must be directly inside
- Missing model = app will crash on launch
- Large models (~100MB+) will slow down app startup


DEPENDENCIES WITH NATIVE CODE
------------------------------

These packages require rebuilding when updated:
- react-native-vosk
- react-native-fast-tflite
- expo-camera
- expo-dev-client

After updating these, always run:
1. npm install
2. npx expo prebuild --clean
3. Rebuild the app


PERMISSIONS
-----------

⚠️ App requires these permissions to work:
- CAMERA (for turn prediction)
- RECORD_AUDIO (for voice commands)

If either is denied, features will fail.
Always test permission flows after rebuilding.


BUILD PROFILES
--------------

Development:  Requires dev server, has debugging
Preview:      Standalone APK for testing
Production:   Optimized AAB for Play Store

⚠️ Always test with preview builds before production release


GIT & VERSION CONTROL
----------------------

⚠️ DO NOT commit these folders:
- android/
- ios/
- node_modules/

These are generated by prebuild and npm install.
Add them to .gitignore if not already there.


VOSK MODEL IN GIT
-----------------

⚠️ The Vosk model in assets/model-en-us/ is ~40MB

Consider using Git LFS or documenting model download separately
to avoid bloating the repository size.


EXPO SDK UPDATES
-----------------

⚠️ When updating Expo SDK version:
1. Update all expo packages together: npx expo install --fix
2. Check compatibility: npx expo-doctor
3. Run prebuild: npx expo prebuild --clean
4. Test thoroughly before committing

Native modules may break between SDK versions.


TESTING CHECKLIST
-----------------

Before committing changes, test:
□ Voice command "Start" triggers navigation
□ Camera screen loads and captures frames
□ ML model inference runs without errors
□ Permissions prompt correctly
□ App doesn't crash on launch
□ Build completes successfully


TROUBLESHOOTING
---------------

App crashes on launch:
→ Check if Vosk model exists in assets/model-en-us/
→ Verify android/ folder is up to date (run prebuild)

Voice commands not working:
→ Check microphone permission is granted
→ Verify model loaded (check console logs)

Build fails:
→ Run: rm -rf node_modules && npm install
→ Run: npx expo prebuild --clean
→ Check EAS build logs for specific errors
