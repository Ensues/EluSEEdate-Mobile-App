================================================================================
                    DOCUMENTATION UPDATE GUIDE
                  For Future Reference & Maintenance
================================================================================

This document lists all documentation files that need to be updated whenever
changes are made to the codebase. Use this as a checklist to ensure all docs
stay in sync with code changes.

================================================================================
WHEN YOU CHANGE: MODEL CONFIGURATION
================================================================================

If you modify src/config/modelConfig.ts:

1. texts/DATA_DICTIONARY.txt
   - Update MODEL CONFIGURATION section
   - Update DEVICE CONFIGURATION section
   - Update EXPORTED CONSTANTS section
   - Update any affected variable descriptions

2. README.md
   - Update "Model Configuration" table
   - Update "Intent Channels" section if changed
   - Update "Performance Metrics" section if metrics change
   - Update "Target Device" section if device changes

3. MODEL_CONVERSION_GUIDE.md
   - Update if model architecture changes
   - Update if input/output shapes change

PERIODIC MAINTENANCE
================================================================================

Every major release:

1. Review ALL documentation files
2. Update "LAST UPDATED" dates in:
   - texts/DATA_DICTIONARY.txt
   - Other relevant .txt files

4. Check that all file paths in docs are still accurate
5. Update version numbers if applicable

================================================================================
TIPS FOR MAINTAINING DOCUMENTATION
================================================================================

1. Update docs IMMEDIATELY after code changes (not later)
2. Keep descriptions brief but clear
3. Include units (ms, pixels, etc.) for numeric values
4. Document WHY, not just WHAT
5. Note any limitations or known issues
6. Keep the DATA_DICTIONARY organized by file/section
7. Use consistent terminology across all docs
8. Date your updates for tracking
